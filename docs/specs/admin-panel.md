# ТЗ: Админка для управления продавцами и товарами

> Статус: **draft** · создано 2026-06-11 · приоритет: P1

## Зачем

Сейчас добавление продавцов реализовано частично (форма на табе «Статус»: вставил
supplierId → парсер берёт весь его каталог). Но:

1. **Авто-discovery ненадёжен.** `find_wb_sellers` сканирует каталог-эндпоинт,
   который soft-блокируется WAF на большинстве shard'ов (реально работает только
   `electronic58` / смарт-часы). Поэтому продавцов в категориях ноутбуки/смартфоны/
   планшеты автопоиск не находит. Единственный надёжный путь — добавить supplierId
   вручную, зная его с сайта (`/seller/<id>`).
2. **Нельзя добавить отдельный товар.** Только продавца целиком. Если нужен один
   конкретный артикул (не весь каталог) — сейчас никак.
3. **Нет видимости каталога продавца** в UI — сколько у него товаров, какие, когда
   последний раз обновлялись.

Админка решает: один экран, где добавляешь продавцов и/или точечные товары,
видишь их состояние, управляешь активностью.

## Что уже есть (переиспользовать)

- `tracked_suppliers` (id, supplier_id, alias, active, created_at) + CRUD API:
  `GET/POST/DELETE /api/suppliers`, `POST /api/suppliers/{id}/toggle`.
- `db.active_supplier_ids()` — collector читает на каждом тике.
- `watchlist` + `target_prices` — пример точечной работы с nm_id.
- `WbCffi.fetch_json` — валидация supplierId/nm_id живым запросом к WB.

## Объём работ

### 1. Отдельный таб «Админка» (или подраздел в «Статус»)

Вынести управление из общего «Статуса» в отдельный экран `⚙️ Админ`:
- секция «Продавцы»
- секция «Точечные товары»
- секция «Telegram-бот» (переехать сюда из «Статуса»)

### 2. Продавцы — расширение

- **Добавление по URL ИЛИ ID.** Принимать как `250090328`, так и
  `https://www.wildberries.ru/seller/250090328` — парсить id из строки.
- **Валидация при добавлении.** До сохранения дёрнуть `sellers/v4/catalog?supplier=<id>`:
  - если товаров > 0 → показать имя продавца + количество, сохранить.
  - если 0 / 404 → предупредить «продавец пустой или не существует», но дать
    сохранить принудительно (вдруг наполнится позже).
- **Карточка продавца раскрывается** в список его товаров (первые N, с ценой и
  ссылкой). Источник — `products WHERE supplier_id = ?`.
- **Алиас редактируется** инлайн (сейчас задаётся только при создании).
- **Бейдж состояния:** 🟢 активен / 👻 спит (0 товаров) / ○ выключен. (частично
  сделано в UI — фикс 2025-06-11.)
- **Авто-сон:** продавец с 0 товаров N тиков подряд → авто-`active=0` + пометка,
  чтобы не молотить впустую. Хранить счётчик пустых тиков.

### 3. Точечные товары (новое)

Новая сущность — мониторить **конкретный артикул** независимо от продавца.

- Таблица `tracked_items (nm_id PK, alias, added_at, active)`.
- Добавление по `nm_id` или WB-URL (`/catalog/<nm>/detail.aspx`).
- Валидация через `card.wb.ru` / `sellers` — достать имя, бренд, цену.
- Collector в каждом тике, помимо продавцов, опрашивает и эти nm_id
  (батч-запросом `card.wb.ru/cards/v2/detail?nm=...;...`, до 100 за раз).
  ⚠️ `card.wb.ru` сейчас TCP-режется по нашему IP — нужна проверка через VPN,
  возможно fallback на `sellers`-эндпоинт или basket-CDN.
- Эти товары тоже идут в детектор (дропы + целевые цены) и в каталог.

### 4. API (доточить)

```
PATCH  /api/suppliers/{id}        — изменить alias
GET    /api/suppliers/{id}/items  — товары продавца (пагинация)
POST   /api/items                 — добавить точечный товар {nm_id|url}
GET    /api/items                 — список точечных
DELETE /api/items/{nm_id}
POST   /api/resolve               — {input: "url или id"} → {type, id, name, products_n}
                                     универсальный резолвер для формы добавления
```

### 5. UX-мелочи

- Поле ввода понимает и id, и url, и копипаст с пробелами.
- После добавления продавца — кнопка «обновить сейчас» (форс-тик по нему, не
  ждать 30 мин).
- Подтверждение при удалении (есть).
- Тосты на успех/ошибку (есть).

## Не входит (отдельные ТЗ)

- Авторизация/мультипользовательность (сейчас одно устройство, без логина).
- Импорт списка продавцов из CSV.
- Роли (админ/просмотр).

## Оценка

| Блок | Время |
|---|---|
| Таб «Админка» + переезд секций | 1ч |
| Продавцы: URL-parse, валидация, раскрытие каталога, инлайн-алиас | 2-3ч |
| Точечные товары: таблица, API, опрос в коллекторе, детектор | 3-4ч |
| Авто-сон продавцов | 1ч |
| Форс-тик «обновить сейчас» | 1ч |

Итого ~1 рабочий день.

## Открытые вопросы

1. `card.wb.ru` для точечных товаров режется по IP — проверить под VPN,
   иначе точечные товары брать через `sellers`-эндпоинт (но тогда нужен и
   supplierId товара).
2. Форс-тик по одному продавцу — выносить ли коллектор в отдельную таску, чтобы
   ручной триггер не конфликтовал с плановым циклом.
